# 深紅倖存者 R34：效能 Log 匯出修正

## 問題原因

R33 的「下載效能 Log」按鈕在沒有已儲存資料時會被設為 `disabled`。

只開啟「效能監測」只會在畫面上顯示數字，不會把歷史資料寫進 `performanceLog.samples`。因此若沒有先按「效能記錄：開始」，下載按鈕會被停用。停用的 HTML 按鈕不會觸發 click，也不會執行原本的「目前沒有資料」Toast，所以看起來像按了完全沒反應。

另外，R33 只使用 `Blob + <a download>` 下載。若遊戲執行在 sandbox iframe、線上預覽器或限制 Blob 下載的環境，瀏覽器可能阻擋檔案。

## R34 修改

- 下載按鈕永遠可以點擊。
- 沒有資料時會明確提示先開始記錄。
- 只開啟監測但沒有正式錄製時，只要目前已有畫面幀資料，也能匯出一筆 `snapshotOnly` 快照。
- 錄製中按下載時，會先把尚未完成的一段取樣寫入 Log。
- Blob URL 延後 30 秒釋放，下載連結延後移除，改善部分瀏覽器來不及讀取的情況。
- 新增「開啟效能 Log」，在新分頁顯示 JSON，可使用瀏覽器另存新檔。
- 新增「複製效能 Log」，將完整 JSON 複製到剪貼簿。
- 結算畫面也新增「開啟效能 Log」備援。
- 下載失敗會寫入 Console 並顯示錯誤訊息，不再假裝一定成功。

## 建議流程

1. 暫停遊戲。
2. 按「效能記錄：開始」。
3. 繼續遊戲並重現卡頓至少 30 秒。
4. 暫停遊戲。
5. 按「效能記錄：錄製中」停止。
6. 按「下載效能 Log（JSON）」。
7. 若瀏覽器仍阻擋下載，按「開啟效能 Log」後另存，或按「複製效能 Log」。

## 手動修改位置

### 下載按鈕被停用

在 `game.js` 搜尋：

```js
function syncPerformanceLogButtons()
```

R33 的問題行是：

```js
pausePerfDownloadButton.disabled = !hasData;
```

R34 已改為不停用下載按鈕，讓沒有資料時也能顯示原因。

### Blob 下載

在 `game.js` 搜尋：

```js
function downloadPerformanceLog()
```

R34 在這裡加入延後釋放 Blob URL、錯誤處理，以及開啟／複製備援。
