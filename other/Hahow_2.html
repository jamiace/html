Hahow 影音學習 AI 續航模組

跨組織協調與商業級架構之產品開發企劃書

【專案定位與開發原則】
鑑於我方非 Hahow 平台方，且技術由外包廠商開發，本專案放棄「修改 Hahow 核心播放器」與「高頻率即時對話」等高風險作法。產品定位為「低侵入式網頁組件」，核心任務是「防囤課（拉高完課率）」與「微測驗（確保學會）」。開發原則為：Hahow 研發負擔最小化、外包 Token 成本固定化。

一、 產品發想與技術架構

為克服三方跨組織（我方、外包、Hahow）在法務、資安與 API 權限審查上的摩擦力，產品全面改用低度對接的設計：

1. Web 端：「貼行代碼」輕量組件

運作方式： 外包廠商將組件開發完成後託管於我方伺服器。Hahow 研發團隊完全不需要改動影音核心程式，只需在前端網頁貼上一行類似 Google Analytics 的 JavaScript 代碼即可完成部署。

功能機制： 網頁組件自動監聽網頁原生的 HTML5 影片時間軸。當播放進度到達 50% 時，組件自動在側邊欄跳出預先生成的單選微測驗。答錯時，系統直接讀取快取資料提示：「建議回到影片 03:15 處重新觀看。」

2. LINE 端：純推播「防囤課」機器人

運作方式： 放棄高成本、多輪對話的即時互動。當 Hahow 系統偵測到使用者買課後超過 3 天未觀看，僅需透過一個極簡的 Webhook 發送訊號給我方 LINE 伺服器。

功能機制： LINE 自動發送一則 50 字內的預生成「實用痛點金句」或「單元微劇透」，並附上直接喚醒網頁回到該課程單元的超連結，以最低阻力引導學員回看影片。

【Token 成本與算力控管】
LLM 僅在課程測試啟動時，由外包後台跑一次腳本，讀取字幕（SRT）與講義文字，一次性批次生成所有摘要與題目存入資料庫（快取）。使用者前端互動僅為「讀取靜態資料」，AI 呼叫次數與用戶觀看數無關；另針對前端發問設定「單人每日 3 次 Quota 鎖」，將即時算力成本鎖死在可預測的範圍。

二、 跨組織商業合作策略

目標客群 (TA)： 鎖定 Hahow 平台上買了課卻容易囤課、時間破碎化的「三分鐘熱度上班族」。

封閉式 Pilot 測試方案（降低商務阻力）：
不要求 Hahow 開放全站資料庫。提案時由 Hahow 挑選 3 堂最想拉高指標的熱門職場課。由 Hahow 課程 PM 人工下載這 3 堂課的講義 PDF 與字幕文字檔（SRT）提供給我方，再轉交外包廠商。這樣能達成 Hahow 零資安風險、外包廠商開發環境封閉 的安全合作架喚。

三、 專案開發與協調週期 (務實預估：5 ~ 6 個月)

考量跨公司行政流程、合約修訂、規格澄清與三方聯合調試，拉長並梳理出以下務實流程：

第 1 ~ 2 個月 商務對齊與資料準備期（我方主導）
  • 與 Hahow 談定合作意向，確認 Pilot 測試的 3 堂課程，並人工取得講義與字幕資料。
  • 與外包廠商釐清規格，控管外包合約。將技術架構定案為「後台批次預生成」，確認 API 規格書（Spec）與計費防禦機制。
第 3 ~ 4 個月 封閉式研發與後台生成期（外包廠商開發）
  • 外包廠商建立 RAG Pipeline，將 3 堂課資料餵給 LLM 批次生成章節精華、題庫與引導秒數。
  • 外包廠商開發 Web 輕量內嵌代碼與 LINE Bot 基礎接收功能。
  • 品質驗收點（我方審查）： 由我方 PM 人工抽驗外包廠商生出的題庫品質，校正 Prompt 消除幻覺。
第 5 ~ 6 個月 三方聯合測試與正式上線（三方聯調）
  • Hahow 研發團隊於測試環境（Staging）貼上網頁嵌入碼，測試組件監聽影片時間軸的穩定度。
  • 聯調 Hahow 進度事件與我方 LINE 伺服器的 Webhook 接收。
  • 小規模學員封測、排除 Bug，正式投放上線。

四、 商業級預算與成本結構精算

基於標準軟體工程人力市場行情（以 1 PM、1 前端、1 後端/AI 之基本配置，時程 6 個月計算）以及承載萬人級 Hahow 流量之商業雲端架構，預算重新精算如下：

預算項目 估算金額 (NTD) 費用屬性與說明
外包廠商研發與聯調費用 $1,500,000 ~ $2,400,000 一次性開發費用。
依據台灣市場合規軟體外包行情，資深工程師與專案管理每人月（Man-Month）綜合管銷成本約 12 ~ 15 萬元。半年期專案（總計約 12~15 人月）之合理商業報價區間在此範圍。包含跨組織（Hahow）技術規格對接、客製化 RAG 斷點生成腳本、Web 嵌入式 JavaScript 組件及企業級 LINE Webhook 伺服器建置。
商業級雲端基礎建設費 (Cloud Infra) $35,000 ~ $50,000 / 月 持續性營運費用。
此為承載 Hahow 平台高併發流量之標準 AWS / GCP 企業級配置。包含：自動擴展伺服器（Auto-scaling EC2）、高可用性資料庫快取（Redis / Managed MySQL）、靜態資源加速（CDN）、以及符合企業標準的基礎資安防護（WAF、DDoS 防禦與定期備份機制）。
LLM API Token 算力費 $15,000 ~ $30,000 / 月 持續性營運費用（變動成本）。
即使採取「後台批次預生成」以節省算力，當 Pilot 測試擴大至數萬名學員、且每人每日享有 3 次提問 Quota 時，企業級模型（如 GPT-4o 或 Claude 3.5 Sonnet）的即時上下文解析仍會產生相應的 Token 消耗。此處以月活躍用戶（MAU）達 5,000 人進行商業推估。
我方專案管理（PM）內部成本 視公司內部編制而定 負責介接 Hahow 與外包廠商，主導階段二的題庫品質人工抽驗與商務談判。
