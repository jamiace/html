1

YT Player 功能與限制

以下依目前播放器套件與 YouTube API 可支援範圍進行說明。

A畫面放大縮小
可以做

APP 端可針對播放器所在容器或顯示區域進行縮放控制，因此畫面放大、縮小可納入調整。

B影片加速、長押 2 倍播放
目前套件未支援

目前使用的播放器套件未提供影片速度控制與長押 2 倍播放的操作能力，因此無法直接實作。

原因說明：此類操作需要播放器層級支援 playback speed / gesture override；若套件未開放對應控制，APP 端無法穩定介入。
CCC 字幕
目前套件未支援

目前播放器套件未提供 CC 字幕切換或字幕顯示控制介面，因此無法由 APP 端直接控制。

原因說明：字幕功能需由 YouTube Player 或其嵌入套件提供控制參數；若套件未開放，APP 端不能保證字幕狀態與切換體驗。
D全螢幕情緒符號互動
可以做

可以做情緒符號互動，但若要保留可點選的情緒符號 Bar，播放器畫面就不會是完整意義上的全螢幕。

體驗說明：全螢幕時若疊加一條互動 Bar，會佔用畫面空間，可能影響影片觀看沉浸感；此項可做，但建議需確認 UI 呈現方式。
建議處理方式：播放器原生能力不足的項目，建議在需求規格中清楚標示「受播放器套件限制」。可做項目則進一步確認 UI 位置、互動時機與是否影響觀看體驗。

APP 優化項目

以下為 APP 端可進行之調整與目前狀態說明。

A啟動畫面 Loading Bar
可以做

一開始的 Loading Bar 就會開始動，而不是停住後才開始動。此項可進行調整。

B第三方註冊條款勾選 UI
目前已經有做

目前在「第一次」點選第三方登入時，會顯示隱私權條款供使用者勾選；登入過後就不會再特別彈窗提醒。

規則說明：此邏輯與註冊頁一致，僅在註冊或首次第三方登入當下要求勾選條款。
C最新上架影片數與展開
可以做

最新上架區塊可由目前 3 支影片調整為 5 支，並增加「看更多」展開功能。

顯示規則：預設顯示 5 支；點選「看更多」後最多展開至 10 支，排序依最近上架時間。
D最近三支影片播放進度
可以做

手機端可紀錄最近三支影片最後播放的時間，下次進入時可從上次紀錄時間開始播放。

紀錄範圍：目前規劃以最近三支影片為範圍，紀錄最後播放時間點。

額外想法：YT 底下的 AI 測驗與量子幣

以下說明 YouTube API 對測驗串接的限制。

AYT 底下的 AI 測驗能不能做？能不能連上量子幣？
YouTube API 未提供對應入口

經確認，YouTube API 沒有提供測驗的連結或入口，因此無法由 APP 端直接串接 YouTube 底下的 AI 測驗，也無法直接與量子幣連動。

補充說明：若後續需要「測驗 × 量子幣」機制，建議改以 APP 內建測驗模組方式規劃，測驗題目、完成條件與量子幣發放規則可由 APP / 後台自行控管。
